# Go + PostgreSQL Web App with sqlc

Production-ready web application using **sqlc** for type-safe database queries.

## Why sqlc?

| Benefit | Description |
|---------|-------------|
| **Type Safety** | SQL errors caught at compile time, not runtime |
| **Performance** | Uses pgx native driver (fastest) |
| **Maintainability** | SQL in `.sql` files, easy to review and test |
| **IDE Support** | Full autocomplete on generated Go code |
| **No Reflection** | Unlike GORM/sqlx, zero runtime overhead |

## Quick Start

```bash
# 1. Install sqlc
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest

# 2. Start PostgreSQL
make db-up

# 3. Run migrations
make db-migrate

# 4. Generate Go code from SQL
make generate

# 5. Run server
make run
```

## How sqlc Works

### 1. Write SQL queries with annotations

```sql
-- queries/users.sql

-- name: GetUser :one
SELECT * FROM users WHERE id = $1;

-- name: ListUsers :many
SELECT * FROM users WHERE active = $1 ORDER BY name;

-- name: CreateUser :one
INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *;
```

### 2. Run `sqlc generate`

This creates type-safe Go code:

```go
// Generated by sqlc
func (q *Queries) GetUser(ctx context.Context, id int32) (User, error)
func (q *Queries) ListUsers(ctx context.Context, active bool) ([]User, error)
func (q *Queries) CreateUser(ctx context.Context, arg CreateUserParams) (User, error)
```

### 3. Use in your application

```go
queries := db.New(pool)

// Fully typed! Compiler catches errors
user, err := queries.GetUser(ctx, 1)

// Wrong type? Won't compile!
// user, err := queries.GetUser(ctx, "1")  // вқҢ Compile error
```

## Transactions with sqlc

```go
tx, err := pool.Begin(ctx)
defer tx.Rollback(ctx)

// Use queries with transaction
qtx := queries.WithTx(tx)

user, _ := qtx.CreateUser(ctx, ...)
post, _ := qtx.CreatePost(ctx, ...)

tx.Commit(ctx)
```
